
## 预处理思路

1. **初步过滤和实体提取**：
   - 使用大模型对所有diff文件进行初步过滤，排除掉很小的变动。
   - 对于较大的变动，从diff文件中提取出代码实体（如函数名、类名）。
   - 返回一个JSON对象，表示每个diff文件的路径和实体的映射。
   - 基于过滤后的diff文件，确定最小根目录。

2. **知识图谱构建 && 依赖关系图构建**：
   - 从Pull Request的仓库和分支克隆项目。
   - 构建项目的代码知识图谱，包含所有代码实体，且携带各个实体的上下文。
   - 使用`dependency-cruiser`对最小根目录进行依赖分析，生成依赖关系图。

> 获取到的有：所有 diff 文件代码、文件路径, 剩余 diff 代码的每个 diff 文件的实体数组, 最小根目录, 代码知识图谱, 依赖关系图

3. **依赖分析 & 划分模块处理**：
   - 基于依赖关系图，将遗留的diff文件分为多个组，每个组代表一个模块。
   - 获取模块内所有diff文件的实体数组。
   - 基于实体数组在代码知识图谱中检索相关上下文信息

4. **Code Review**：
   - 将每个模块的上下文信息和模块内的所有diff文件整合在一起，提供给大模型进行Code Review，生成审查报告和建议。
